What happens when you initialize a repository? 

a .git folder is created that allows us to treat that directory as a repository.

Why do you need to do it?

So you can use your files as a repository and use version control

How is the staging area different from the working directory and the repository?

The stagging are is the area you add files before committing, so you don't have to commit all the files you have.

What value do you think it offers?

Lets you choose what files do you want to commit instead of having to commit all.

How can you use the staging area to make sure you have one commit per logical change?

Seeing what's in the staging area and adding only what you want to commit will let you commit only those files that compound one logical change.

What are some situations when branches would be helpful in keeping your history organized? How would branches help?

When you want to make a version of your game changing some things, like language or difficulty.
When you want to try out a feature that you don't know if it'll work but you want to keep the working version the same way you currently have it.
